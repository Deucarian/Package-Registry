# Deucarian Portfolio Migration

Schema version: 2

This ledger records the coordinated governance rollout. It is hand-maintained and is never generated by the organization audit.

## Release A: compatibility bridge

1. Release Package Installer and Bootstrap readers that prefer canonical `kind` and `groupId` while accepting legacy-only catalogs.
2. Publish Package Registry schema v2 with canonical domain groups and artifact kinds.
3. Retain unchanged `category`, `type`, and `ecosystemGroup` projections for this tagged release.
4. Repair exact dependency-version drift, sample contracts, and direct dependency declarations in dependency order.
5. Regenerate and validate both consumer fallback catalogs from `packages.json`.

## Release B: v2 only

1. Verify the released Installer and Bootstrap consumed the bridged registry successfully.
2. Remove the three legacy projection fields and their reader fallbacks.
3. Keep package IDs, display names, repositories, and Git channel URLs unchanged.

## Refactoring boundary

- No package merger, split, rename, or new repository is approved by this migration.
- Generated clone and dependency reports are evidence only.
- Accepted or rejected extractions live in the reviewed refactoring decision ledger.
- Logging's Editor dependency remains during this program and is tracked as a focused compatibility follow-up.
