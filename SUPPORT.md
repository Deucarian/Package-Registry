# Support for the Deucarian Package Registry

Use [GitHub Issues](https://github.com/Deucarian/Package-Registry/issues) for reproducible catalog, governance metadata, validation-tooling, generated-audit, or documentation defects. Search existing issues before opening a new one.

Include the affected commit, command and Python version, relevant package IDs, the smallest input that reproduces the problem, expected and actual output, and a sanitized traceback. State whether generated artifacts were produced with authoritative or local inputs.

`main` is the stable registry consumed by the stable package channel. `develop` is preview governance and may change before promotion.

Package-specific runtime and editor defects belong in the package's own repository. Installer behavior belongs in [Deucarian Package Installer](https://github.com/Deucarian/Package-Installer/issues).

Do not disclose vulnerabilities, private repository inventories, credentials, or personal data in an issue. Follow [SECURITY.md](SECURITY.md) for private security reports. Support is provided on a best-effort basis.
